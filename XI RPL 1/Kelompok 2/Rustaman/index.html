git revert

Perintah untuk membalikkan efek suatu commit dengan membuat commit baru. Tidak menghapus atau menulis ulang riwayat; aman dipakai setelah perubahan dipublikasikan.

Dasar

Definisi

Membuat commit pembalik yang menambahkan kebalikan patch commit target.

Bedakan

vs reset

reset memindah pointer branch; bisa menghapus jejak. revert menambah jejak baru.

Tim

Kolaborasi

Tidak memaksa force push; histori tetap dapat diaudit.

Granular

Selektif

Bebas pilih commit lama/tengah/head tanpa memengaruhi commit lain.

Konsep Inti git revert

Alih‑alih menghapus commit salah, kita tambahkan commit baru yang membatalkan patch. Ini menjaga histori kronologis sehingga investigasi masa depan tetap transparan. Setiap revert adalah catatan eksplisit "apa" dan "mengapa".

revert

Mencatat rollback eksplisit; histori lengkap utuh. Baik untuk branch publik.

git revert <hash>
reset

Menulis ulang pointer; rawan konflik kolaborasi jika sudah dipush. Hindari --hard di branch bersama.

git reset --hard <hash>

1. Temukan

Lihat daftar singkat. git log --oneline

2. Revert tunggal

Balik satu commit. git revert a1b2c3d

3. Konflik

Selesaikan manual lalu git add . & git revert --continue

4. Banyak

Rentang terakhir 3: git revert HEAD~3..HEAD

5. Merge

Revert merge: git revert -m 1 <merge_hash> (1=parent utama)

Sintaks & Contoh

Revert dasar: git revert <hash>. Banyak hash berurutan: jalankan berantai atau gunakan rentang. Komit non-berurutan: sebutkan satu per satu. Membatalkan revert (membalik revert) cukup revert commit revert tersebut.

Revert Merge & Konflik

Saat revert merge, git butuh tahu parent mana yang dianggap baseline: gunakan -m 1 (parent pertama). Konflik umum: file sudah berubah setelah commit target. Strategi: fokuskan patch; bila patch besar dan banyak noise formatting, pertimbangkan refactor terpisah sebelum revert.

Best Practice & Anti-Pattern

Gunakan revert untuk: rollback aman setelah deploy, membatalkan fitur eksperimental, audit jelas. Hindari: menumpuk revert atas revert tanpa perbaikan akar, mengandalkan revert untuk uji coba yang mestinya pakai feature flag. Format pesan: revert: batalkan fitur pencarian cepat (#123).

Ringkasan Cepat

Kapan

Bug pasca push, rollback parsial, butuh jejak jelas.

Plus

Tidak rewrite sejarah, mudah audit, minim risiko tim.

Minus

Histori bisa ramai, konflik saat basis sudah banyak berubah.

Tips

Commit kecil fokus; hindari gabung refactor + fitur; review sebelum revert.
